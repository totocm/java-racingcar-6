## 🚀 N 차 리드미
## 1차
## 공통 : 사용자가 잘못된 값을 입력하면 Exception 발생
--------------------------------------- 
#### ----------------------------------
# 어떻게 만드는지 구체적인 설명은 해보면서 테스트하면서 진행해야하므로
# 진행 방향에 대해서만 리드미에 남기겠습니다.

#### 1번
# 입력을 받으면
# !빈 문자 예외를 우선 터트리고
# 통과하면 입력받은 값을 리스트<객체>에 담급니다
# !forEach를 통해서 순회하며 길이가 5 초과하면 예외를 또 터트려줍니다

#### 2번
# 게임을 시작합니다.
# 사용자가 몇 번 게임을 진행시킬 지 입력 받습니다.
# ! 빈 문자면 예외를 터트립니다.
# 파스인트를 사용하여 형변환을 한 뒤.. 숫자를 활용함
# ! 파쓰인트로 받았을 때 문자열 예외에 대한 조사 -> NumberFormatException 자동생성
# IllegalArgumentException, Exception, NumberFormatException 의 레벨을 알아보고
# 레벨 순서대로 작성

#### 3번
# 반복문을 사용하여 리스트에 담긴 객체를 꺼내와서 경주를 진행시킵니다.
# 위에서 받은 파쓰인트 Number을 활용하여 랜덤 출력이 4 이상인 경우
# if (RandomNumber >= 4) { car.setMove(car.getMove() + 1)) } 를 해주면
# 처음엔 문자 "-" 를 더해줄려고 했지만 숫자가 더 간편할 것으로 보이네요.

#### 4번
# 레이싱 객체를 만들까 생각도 들지만 초 간단 이라는 항목을 생각하여
# 선을 넘지 않는 java-two 입니다.
# int MaxMove = -1; (만약 아무도 움직이지 못했을 경우를 대비하여? )
# forEach로 순회하면서 MaxMove의 값을 변경해주고
# forEach를 한번 더 순회하여 MaxMove의 값을 가진 유저를 찾아서
# car 에 toString 을 작성하여 대괄호를 빼주고
# 객체들 리스트를 만들어 담은 뒤 그 값을 꺼내준다 - 1차 리드미 실행방법
